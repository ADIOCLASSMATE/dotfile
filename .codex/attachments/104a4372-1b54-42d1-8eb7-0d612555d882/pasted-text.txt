# 一、论文数量的硬约束

## 1. Anchor paper

全文最多选择 12 篇 anchor paper。

每条技术谱系：

- 最多深入讲 2 篇；
- 可以只深入讲 1 篇；

- 同一篇论文只能有一个主要讲解位置，其他章节使用交叉引用，不得重复完整介绍。

Anchor paper 才需要完整分析：

- 模型；
- 核心公式；
- loss；
- 训练算法；
- 采样算法；
- repo 实现；
- 前后工作关系；

Bridge paper 只用于解释两项 anchor paper 之间发生了什么。

- 或比较表中的一行；

- 独立占用大型二级章节；
- 展开所有实验；
- 罗列全部公式；
- 深入阅读整个 repo；
- 写成与 anchor paper 同等篇幅。

## 3. Extended paper

其他已下载论文统一放入：

- “延伸阅读”表；

# 二、这次重写要解决的核心问题

当前初稿的问题不是论文数量不足，而是：

1. 论文按照类别连续罗列，读者难以感受到“为什么下一项工作会出现”；

# 三、总体叙事结构：五类作为坐标，七条谱系作为正文主线

保留原有五类分类，但将其降为“横向分类坐标”，不要继续把五类直接作为全文唯一的章节结构。

五类分类用于回答“这篇论文处在哪个状态空间和生成结构中”：

1. 离散文本，非左到右的自回归或顺序灵活建模；
2. 离散文本，双向并行 NAR 与 iterative refinement；
3. 离散文本，离散状态 diffusion / masked diffusion / discrete flow；
4. 离散文本，在连续 token-level 状态或连续松弛上建模；
5. continuous embedding / learned latent / flow language modeling。

正文则必须以以下七条技术谱系为主线：

## 谱系 1

AR  
→ permutation / order-flexible AR  
→ XLNet、Insertion Transformer

核心问题：

- 生成顺序是否必须等于文本表面顺序？
- 改变顺序是否等于摆脱自回归？
- 为什么给定 permutation 或 insertion action trajectory 后仍然是 AR？

## 谱系 2

one-shot NAT  
→ iterative refinement  
→ CMLM / Mask-Predict

核心问题：

- 一次性独立预测为什么导致 multi-modality？
- fertility、length prediction 和 knowledge distillation 在解决什么？
- iterative refinement 如何用多轮条件更新修补 one-shot NAT？
- CMLM 与普通 MLM 有什么不同？
- Mask-Predict 的训练和推理为什么并不完全一致？

## 谱系 3

CMLM / masked iterative generation  
→ denoising bridge  
→ masked diffusion

重点工作：

- SUNDAE；
- DiffusionBERT；
- D3PM；
- MDLM。

核心问题：

- 什么时候“反复 mask-predict”开始变成有概率语义的 diffusion？
- 普通 MLM objective 与 diffusion ELBO 有何区别？
- forward corruption、reverse transition 和 time conditioning 分别增加了什么？
- SUNDAE、DiffusionBERT 应被称为直接前驱、概念桥梁还是并行路线？

注意：必须核验真实年份和方法关系。不要为了线性故事而错误声称 D3PM 一定直接继承 SUNDAE，或者 MDLM 直接继承 DiffusionBERT。允许画成分支汇合图。

## 谱系 4

D3PM  
→ SEDD / MDLM  
→ LLaDA

核心问题：

- D3PM 如何用 transition matrix 建立通用离散 diffusion？
- SEDD 为什么改为估计 probability ratio / discrete score？
- MDLM 如何利用 absorbing mask 和 SUBS 参数化化简目标？
- LLaDA 如何把 masked diffusion recipe 扩展到大规模预训练和 SFT？
- 从通用离散状态扩散到大规模语言模型，真正需要解决的是什么？

注意：SEDD 与 MDLM 更可能是从 D3PM/离散扩散出发的两条并行理论路线，不要强行写成 SEDD → MDLM。

## 谱系 5

MDLM / large masked DLM  
→ DiffuLLaMA / DiffuGPT  
→ Block Diffusion

这里实际是两条分支：

1. 复用 AR checkpoint 的 adaptation 路线；
2. block/semi-autoregressive 路线。

核心问题：

- DLM 是否必须从零训练？
- AR objective 和 masked diffusion objective 有什么可连接之处？
- 为什么 full-sequence diffusion 难以使用标准 KV cache？
- Block Diffusion 如何在 block 之间恢复 AR、block 内保留 diffusion？
- block size 为 1 时是否恢复标准 AR？
- block size 为全序列时是否接近 full diffusion？
- 这种折中换来了什么，又失去了什么？

## 谱系 6

Diffusion-LM  
→ SSD-LM / CDCD / RDLM  
→ Flow Map Language Models

核心问题：

- 为什么早期连续文本 diffusion 选择 token embedding？
- embedding rounding 为什么成为瓶颈？
- simplex/logit/one-hot 表示比普通 embedding 改善了什么？
- RDLM 为什么强调 categorical geometry 和 statistical manifold？
- diffusion trajectory 与 flow matching trajectory 有何区别？
- Flow Map LM 如何把多步 flow 压缩到 one-step 或 few-step map？
- one-step 生成是否真的消除了训练和推理成本？

## 谱系 7

Autoencoder latent  
→ Latent Diffusion for Language Generation  
→ Cola DLM / TextLDM / LDLM

以及一条汇合但不一定直接继承的分支：

token/contextual embedding flow  
→ ELF

核心问题：

- token-aligned continuous state 与 learned semantic latent 的区别是什么？
- VAE/autoencoder 为什么能够改变生成对象？
- reconstruction fidelity 为什么不等于 latent 易于 diffusion？
- Cola DLM、TextLDM、LDLM 分别如何处理 latent prior、表示对齐和联合训练？
- ELF 是否真的属于 autoencoder latent 路线？
- ELF 更适合被解释为 latent 路线的延续，还是 embedding flow 的独立汇合点？

不要强行写成：

Autoencoder → Cola → TextLDM → LDLM → ELF

除非本地论文明确支持这种直接继承关系。

更准确的叙述可以是：

- Autoencoder latent → Cola/TextLDM/LDLM；
- embedding-space continuous flow → ELF；
- 两条路线在“将大部分生成过程移出离散 token space”这一目标上汇合。

# 四、箭头的含义必须严格区分

全文中的箭头不能都被写成“后者直接建立在前者上”。

为每一条关系选择以下标签之一：

- `direct extension`：论文明确基于前一工作；
- `shared formulation`：共享相同数学形式，但不一定直接继承；
- `conceptual precursor`：前一工作提供概念基础；
- `parallel branch`：同期或相近时期的独立路线；
- `retrospective bridge`：后来看起来连接两种范式，但历史上不一定直接承接；
- `contrast`：用于对比而非继承；
- `convergence`：不同路线最后汇合到相似目标。

在每个谱系章节中至少加入一张“关系表”：

| 从 | 到 | 关系类型 | 继承内容 | 改变内容 | 解决的问题 | 新增代价 |
|---|---|---|---|---|---|---|

不得仅凭发布时间接近就声称存在继承关系。

## 2. 统一符号与数学基础

本章建立统一 notation，后文不再随意改变符号。

至少定义：

- 离散序列：
  \(x=(x_1,\ldots,x_n)\)；
- 词表：
  \(\mathcal V\)；
- 顺序：
  \(\pi\)；
- mask 集合：
  \(M\)；
- clean state：
  \(x_0\)；
- noisy state：
  \(x_t\)；
- 连续表示：
  \(z_t\)；
- transition matrix：
  \(Q_t\)；
- cumulative transition：
  \(\bar Q_t\)；
- encoder、decoder、denoiser、velocity field；
- block 和 block index。

本章必须解释而不是只罗列以下公式：

### 2.1 左到右 AR

\[
p_\theta(x)=\prod_{i=1}^{n}p_\theta(x_i\mid x_{<i})
\]

说明：

- chain rule；
- teacher forcing；
- NLL；
- 推理串行深度；
- KV cache。

### 2.2 任意顺序 AR

\[
p_\theta(x\mid \pi)
=
\prod_{t=1}^{n}
p_\theta(x_{\pi_t}\mid x_{\pi_{<t}})
\]

解释：

- permutation 改变条件顺序；
- 为什么仍是 AR；
- 对 permutation 取期望的含义；
- 训练顺序和真实生成顺序的区别。

### 2.3 one-shot NAT

\[
p_\theta(y\mid x,N)
=
p_\theta(N\mid x)
\prod_{i=1}^{N}p_\theta(y_i\mid x,N)
\]

解释：

- 条件独立；
- multi-modality；
- length/fertility；
- knowledge distillation 为什么有效。

### 2.4 MLM/CMLM

\[
\mathcal L_{\mathrm{CMLM}}
=
-\mathbb E_{M}
\sum_{i\in M}
\log p_\theta(y_i\mid x,y_{\bar M})
\]

解释：

- 这通常训练的是条件分布族；
- 它是否定义一致的 joint distribution；
- 为什么加上 iterative sampler 后才成为生成算法；
- 它与 masked diffusion 的区别。

### 2.5 离散 diffusion

\[
q(x_t\mid x_{t-1})
=
\operatorname{Cat}(x_t;x_{t-1}Q_t)
\]

\[
q(x_t\mid x_0)
=
\operatorname{Cat}(x_t;x_0\bar Q_t)
\]

继续推导或说明：

- \(\bar Q_t=Q_1Q_2\cdots Q_t\)；
- \(q(x_{t-1}\mid x_t,x_0)\)；
- reverse model；
- ELBO；
- \(x_0\)-parameterization；
- absorbing mask 特例。

### 2.6 continuous-time discrete diffusion

解释：

- rate matrix；
- CTMC；
- reverse rate；
- probability ratio / discrete score；
- tau-leaping。

### 2.7 连续 diffusion

\[
z_t=\alpha_t z_0+\sigma_t\epsilon
\]

解释：

- noise prediction；
- \(z_0\)-prediction；
- score prediction；
- velocity prediction；
- 离散化接口。

### 2.8 Flow Matching

解释并写出：

- probability path；
- conditional path；
- conditional velocity；
- marginal velocity；
- flow matching loss；
- ODE sampling。

不要默认所有论文都使用完全相同的直线路径。每篇论文以其原文为准。

### 2.9 VAE / latent language modeling

写出：

\[
q_\phi(z\mid x),\qquad
p_\psi(x\mid z)
\]

以及适用时的 ELBO：

\[
\mathcal L_{\mathrm{VAE}}
=
-\mathbb E_{q_\phi(z\mid x)}
\log p_\psi(x\mid z)
+
\beta D_{\mathrm{KL}}
\left(
q_\phi(z\mid x)\|p(z)
\right)
\]

再解释：

- reconstruction；
- KL；
- latent prior；
- latent diffusion/flow；
- representation alignment；
- posterior collapse；
- latent geometry。
